# 禁止行为（红线）

## 代码层面

1. **禁止绕过 repository 直接操作数据库** — 路由层不得直接调用 `prisma` 客户端
2. **禁止跨模块无封装调用** — 前端不得 import 后端文件，后端各层不得反向依赖
3. **禁止使用 `any` 类型** — 使用 `unknown` + 类型守卫替代
4. **禁止省略函数注释** — 所有导出函数必须有 JSDoc 注释
5. **禁止修改 `src/generated/` 下的 Prisma 生成代码**
6. **禁止手动修改 `prisma/migrations/` 下的迁移文件**

## 文档层面

7. **禁止不更新文档直接提交代码** — API/路由/结构变更必须同步更新 README
8. **禁止删除 README 固定板块** — 技术栈、API 接口、后续优化项等板块必须保留
9. **禁止私自重构底层架构不做说明** — 重构前必须在本次对话中说明变更影响

## 流程层面

10. **禁止一次性混改多无关模块** — 一个 commit 只做一件事
11. **禁止忽略历史待优化项** — 开发前先查看 README「后续优化项」，避免重复踩坑
12. **禁止引入新的第三方依赖不说明理由** — 新增依赖需在对话中说明必要性

## 受保护文件

以下文件修改前必须在对话中说明原因：
- `backend/prisma/schema.prisma`
- `backend/src/app.ts`
- `frontend/src/App.tsx`
- `package.json`（根级和子级）
- `tsconfig.json`（根级和子级）
 ## 文档同步触发条件

  以下操作必须同步更新 README.md：
  1. 新增或删除 API 接口 → 更新「API 接口」章节
  2. 新增或删除页面路由 → 更新「页面路由」章节
  3. 修改环境变量 → 更新「环境变量」章节
  4. 修改目录结构 → 更新「项目结构」章节
  5. 修改启动/构建命令 → 更新「快速开始」章节

  ## 后续优化项维护

  - 新发现的优化需求追加到对应分类下，格式：`- [ ] 描述`
  - 已完成的项改为 `- [x] 描述`（不删除，保留历史记录）
  - 分类：功能完善 / 用户体验 / 安全加固 / 工程化 / 性能优化 / 内容扩展
  - 每次开发完成后至少补充 1 条后续优化点
  - 架构大幅调整时同步精简更新根目录 CLAUDE.md
